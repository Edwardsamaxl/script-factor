## Step 4.1：演示路径验证（0-2 小时）

**这是黑客松第二重要的事（第一是代码能跑）。**

```
演示路径检查：
□ 打开链接到看到首屏 < 3 秒
□ 用户不需要任何引导就能开始操作
□ 核心 AI 能力 < 5 秒给出反馈
□ 演示路径上每个按钮都可以点
□ 没有明显的视觉 bug
□ 手机端体验正常（必须测试）
□ 加载状态动画正常
□ 错误提示友好（"网络开小差了，重试？"）
```

### RAG 场景额外检查

```
□ 检索结果在 2 秒内返回
□ Rerank 后 Top-3 结果相关度高
□ 生成回答有引用来源（点击可跳转）
□ 流式输出正常（打字机效果）
```

### CV 检测场景额外检查

```
□ YOLOv8 推理 < 2 秒（Modal GPU）
□ 检测结果标注框正确显示
□ 置信度阈值展示合理
□ 支持图片拖拽上传
```

## Step 4.2：性能优化（2-6 小时）

| 优化点 | 方法 | 目标 |
|--------|------|------|
| 首屏加载 | Next.js SSG + 骨架屏 | < 2s |
| AI 响应 | 流式输出（Streaming）| 感知 < 1s |
| 数据库查询 | 索引 + Redis 缓存 | p99 < 200ms |
| 检索延迟 | Qdrant 向量索引优化 | < 500ms |
| 图片加载 | WebP + CDN | < 500ms |
| 冷启动 | Railway 保持活跃 / UptimeRobot ping | 无冷启动 |

## Step 4.3：UI/UX 打磨清单

