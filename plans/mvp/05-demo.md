## 演示脚本结构（30 分钟）

```
[0:00-0:30]  开场 — 一句话解释你做了什么
             "我们做了一个____，让____能够____"

[0:30-1:30]  演示 — 现场操作核心功能
             只演示 P0 功能，不要炫技

[1:30-2:30]  亮点 — 为什么这个想法有意思
             - 用户故事（给谁用，解决了什么问题）
             - 技术亮点（1 个就够了，不要讲架构）
             - 评委可能问的技术问题提前准备

[2:30-3:00]  收尾 — 结束语 + 留一个钩子
             "我们接下来想做____"
```

### 评委高频问题准备

| 问题 | 回答方向 |
|------|---------|
| "技术难点在哪？" | 说 RAG 检索优化 或 YOLOv8 部署经验 |
| "为什么不用 XX 方案？" | 诚实说权衡，现场没时间展开 |
| "怎么做商业化？" | 1-2 句话点出商业模式 |
| "团队怎么分工？" | 独立开发者：强调全栈能力 |

## 备用方案（必须准备）

```
Plan A: 现场网络演示（公网 URL）
Plan B: 录屏播放（提前录好 1080p 60fps，< 3 分钟）
Plan C: 只讲方案不改代码（最坏情况，静态 PPT）
```

**关键保险操作**：
- 提前在另一台设备登录好账号
- 提前录好完整演示视频（带配音最佳）
- 代码 push 到 GitHub，另一台设备可拉取
- 手机开热点，备用网络
