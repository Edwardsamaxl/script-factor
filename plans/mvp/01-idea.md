## Step 1.1：选题决策（0-20 分钟）

### 快速评估矩阵

| 维度 | 权重 | 评估问题 |
|------|------|---------|
| **新颖性** | 20% | 这个想法在目标赛道是新的吗？ |
| **可演示性** | 25% | 评委看到第 5 秒能理解吗？ |
| **技术可行性** | 25% | 24-48h 能做出可运行版本吗？ |
| **影响力** | 20% | 如果成了，用户真的会去用吗？ |
| **个人优势** | 10% | 你/队友有相关经验吗？ |

**操作**：
1. 头脑风暴列出 3-5 个想法
2. 每个想法过一遍上表，匿名评分
3. 选最高分；分数接近则选**最新奇**的

### 技术栈快速筛选

```
涉及这些方向？→ 技术可行性极高，放心选
├── RAG / 知识库问答 → LangGraph + 混合检索链路可直接复用
├── CV / 目标检测 → YOLOv8 + FastAPI 快速搭推理服务
├── 多模态 → Claude Vision / GPT-4o Vision + RAG 架构
├── 数据分析 / 报表 → FastAPI + MySQL + Next.js
└── Agent 自动化 → LangGraph Coordinator + tools
```

## Step 1.2：用户旅程画布（20-40 分钟）

```
[用户入口] → [第一个动作] → [反馈循环] → [核心价值交付] → [传播/留存触发]
    ↓              ↓              ↓              ↓                  ↓
  平台入口     滑动/点击      即时视觉反馈    获得某种"奖励"       分享或再来的钩子
```

**填完这张图，你就知道 MVP 需要哪些功能了。**

### 示例

**RAG 场景**：`[上传文档/提问] → [LLM 理解意图] → [混合检索 Top-K] → [Rerank 精排] → [生成回答 + 引用来源] → [用户点击验证] → [分享/再问]`

**CV 检测场景**：`[上传图片/拍照] → [YOLO 推理 <2s] → [可视化标注] → [导出报告/分享] → [再检测一张]`

## Step 1.3：PRD 填写 + mentor 检查（40-60 分钟）

完成 `PRD.md` 前 3 章，然后找 mentor 排 5 分钟队，只问一个问题：**"这个想法最大的风险是什么？"**，根据回答砍掉 1-2 个 P1 功能。

### PRD 快速填写

```markdown
## 1. 一句话定位
格式：我们为 [目标用户] 提供了 [核心价值]，通过 [关键机制]。

## 2. 核心问题
- 问题 1：[具体描述]
- 问题 2：[具体描述]

## 3. MVP 功能清单
### P0 — 门控（无则项目无意义）
| 功能 | 描述 | 完成标准 |
|------|------|---------|
| F01 | _________ | 可运行 / 可演示 |

### P1 — 重要（可砍）
| 功能 | 描述 | 完成标准 |
|------|------|---------|
| F11 | _________ | 功能完整 |

### P2 — 惊喜（不做）
| 功能 | 描述 |
|------|------|
| F21 | _________ |
```
